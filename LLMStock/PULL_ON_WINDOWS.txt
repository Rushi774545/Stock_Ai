Git pull / merge on Windows (db.sqlite3, chroma, conflicts)
================================================================================

WHY IT HAPPENS
- "Unlink of LLMStock/db.sqlite3 failed" = Git cannot replace/delete the file
  because another program has it open (almost always: Django runserver, or a
  SQLite GUI, or sometimes antivirus indexing).
- chroma.sqlite3 "modify/delete" = your branch still had the file tracked or
  modified locally; upstream removed it from Git (it is local-only now).
- requirements.txt / settings.py = normal text conflicts; edit by hand once.

--------------------------------------------------------------------------------
BEFORE EVERY PULL (recommended)
--------------------------------------------------------------------------------
1) Stop the backend: in the terminal running Django, press Ctrl+C.
2) Close DB Browser / any tool that opened LLMStock\db.sqlite3 or chroma.
3) Pull again:  git pull origin main

--------------------------------------------------------------------------------
If you already started a merge and it failed
--------------------------------------------------------------------------------
1) Stop Django (Ctrl+C).
2) Optional: copy LLMStock\db.sqlite3 somewhere safe if you need the data.
3) If Git still cannot unlink db.sqlite3, rename it (Explorer or PowerShell):
     ren LLMStock\db.sqlite3 db.sqlite3.bak
4) For chroma "deleted vs modified" — accept removal from the repo (file can
   stay on disk for your app; it is gitignored):
     git rm -f LLMStock/chroma_db/chroma.sqlite3
   If that path is wrong, use:  git status   and remove the path Git shows.
5) Resolve text conflicts:
   - Open LLMStock/requirements.txt and LLMStock/stock_platform/settings.py
   - Delete all lines like:  <<<<<<<   =======   >>>>>>>
   - Keep one sensible combined result (all dependencies; one CORS block).
6) Stage and finish:
     git add LLMStock/requirements.txt LLMStock/stock_platform/settings.py
     git add -A
     git commit -m "Merge main: resolve Windows DB paths and text conflicts"
7) Recreate DB if needed:
     cd LLMStock
     python manage.py migrate

--------------------------------------------------------------------------------
If you want to abort the whole merge
--------------------------------------------------------------------------------
     git merge --abort
Then stop Django, rename/remove db.sqlite3 if unlink keeps failing, and pull again.

--------------------------------------------------------------------------------
Fresh clone (no local data to keep)
--------------------------------------------------------------------------------
Easiest way to avoid old tracked SQLite history: clone the repo again into a
new folder, copy only your .env into LLMStock\, run migrate.
